# Security Policy

## Supported Versions

| Version     | Supported          |
| ----------- | ------------------ |
| 0.1.x-alpha | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability within Strata, please report it responsibly:

1. **Do NOT** open a public GitHub issue
2. Email security concerns to the maintainers via the repository's security advisory feature
3. Use [GitHub's private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories) (preferred)

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact assessment
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 1 week
- **Fix Release**: Within 2 weeks for critical issues

## Scope

This policy covers:

- The `@siwon-dev-npm/strata` npm package
- Components, tokens, and utilities in `src/`
- Build scripts in `scripts/`

### Out of Scope

- Demo applications in `src/demos/`
- Storybook configuration
- Development dependencies
